# Contributing to CodeSync

First off, thank you for considering contributing to CodeSync! It's people like you that make open source such a great community.

## How Can I Contribute?

### Reporting Bugs
- Make sure you are on the latest version.
- Open an issue and provide a clear, detailed description of the bug.
- Include steps to reproduce the issue.

### Suggesting Enhancements
- Open an issue and explain your feature request.
- Provide examples of how it would be used and why it would be beneficial.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code adheres to the existing styling.
5. Issue that pull request!

## Local Development
To set up the project locally:
1. Clone the repo: `git clone https://github.com/0007aadil/CodeSync.git`
2. Install dependencies: `npm run install:all`
3. Copy `.env.example` to `.env` in both `client` and `server` folders.
4. Start the dev server: `npm run dev`

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
